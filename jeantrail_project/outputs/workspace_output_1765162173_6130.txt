# JeanTrail Browser

An advanced multi-workspace browser built with Tauri + React + Rust, featuring AI-powered assistance, split-screen layouts, and comprehensive service integration.

## Features

### Core Browser
- **Multi-Tab System**: Four distinct work zones (Local Device, Proxy Network, Standard Web, Mobile Apps)
- **Split-Screen Layouts**: Combine up to 3 panes with different view types
- **Workspace Management**: Save and restore layout configurations
- **RTL Support**: Full right-to-left language support

### Jean AI Assistant
- **Real AI Integration**: Local LLM or cloud-based AI endpoints
- **Action Control**: Jean can navigate, open tabs, and perform browser actions
- **User Confirmation**: All actions require explicit user approval
- **Speech Support**: Placeholder hooks for STT/TTS integration

### Service Integration
- **E-commerce**: Product catalog, shopping cart, order management
- **Job Board**: Job search, applications, career resources
- **Delivery**: Multi-service delivery with real-time tracking
- **Real Estate**: Property listings, virtual tours, mortgage calculator
- **Social Feed**: Posts, stories, user interactions
- **Media Library**: File management, albums, streaming
- **Developer Hub**: Project management, deployments, code analysis
- **Loyalty Wallet**: Rewards program, points management
- **Portfolio Aggregator**: Financial portfolio tracking

## Architecture

### Frontend (React + TypeScript)
- **Components**: Modular React components with Tailwind CSS
- **State Management**: Custom hooks for tabs, Jean AI, workspaces
- **Service Layer**: Clean interface-based service architecture
- **Type Safety**: Comprehensive TypeScript definitions

### Backend (Rust + Axum)
- **Tauri Integration**: Native desktop application wrapper
- **Web Server**: Axum-based HTTP API server
- **Database**: PostgreSQL with Redis caching
- **AI Backend**: Generic LLM abstraction supporting multiple providers

### Database Schema
- **User Management**: Users, profiles, preferences
- **Browser State**: Tabs, workspaces, sessions
- **Service Data**: Extensible schema for all services
- **AI Conversation**: Message history, actions, analytics

## Quick Start

### Prerequisites
- Node.js 18+
- Rust 1.70+
- PostgreSQL 15+
- Redis 7+

### Installation

1. **Clone and Setup**
```bash
git clone <repository-url>
cd jeantrail-browser
npm install
```

2. **Environment Configuration**
```bash
cp .env.example .env
# Edit .env with your configuration
```

3. **Database Setup**
```bash
# Start PostgreSQL and Redis
docker-compose up -d postgres redis

# Run migrations
# (Migrations run automatically via Docker init)
```

4. **Development Server**
```bash
# Start frontend development server
npm run dev

# In another terminal, start Tauri
npm run tauri dev
```

### Configuration Options

#### AI Backend
```bash
# Local LLM (default)
USE_CLOUD_AI=false
LOCAL_LLM_ENDPOINT=http://localhost:8081/v1/chat/completions

# Cloud AI (OpenAI compatible)
USE_CLOUD_AI=true
CLOUD_AI_ENDPOINT=https://api.openai.com/v1/chat/completions
CLOUD_AI_API_KEY=your_api_key_here
```

#### Database
```bash
DATABASE_URL=postgresql://jeantrail:jeantrail123@localhost:5432/jeantrail
REDIS_URL=redis://localhost:6379
```

## Development Workflow

### Adding New Services

1. **Define Service Interface** in `src/services/[service].ts`
2. **Implement Backend API** in `src-tauri/src/[service].rs`
3. **Create Frontend Component** in `src/components/[Service].tsx`
4. **Add Database Schema** in new migration
5. **Update Navigation** in App.tsx

### Building for Production

```bash
# Build frontend
npm run build

# Build Tauri application
npm run tauri build

# Production binaries will be in src-tauri/target/release/bundle/
```

## API Endpoints

### Core APIs
- `GET /health` - Health check
- `POST /ai/generate` - AI generation endpoint
- `GET/POST/PUT/DELETE /api/workspaces/*` - Workspace management
- `GET/POST/DELETE /api/proxy/*` - Proxy network management
